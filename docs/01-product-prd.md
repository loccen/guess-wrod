# 猜不到的词 H5 PRD

## 1. 文档说明

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 猜不到的词 |
| 产品形态 | 移动端 H5 |
| 当前版本 | V0.1 |
| 发布目标 | 可在微信内置浏览器和手机浏览器访问 |
| 核心玩法 | 用户输入猜词，系统返回该词与隐藏答案的语义接近百分比 |

本文档只描述产品范围、页面和验收。评分、接口、数据和架构分别见对应文档。

## 2. 产品目标

### 2.1 首版目标

1. 验证“AI 只返回百分比”的猜词循环是否成立。
2. 验证用户是否愿意连续玩多局随机题。
3. 验证 AI 评分是否比传统相似度更符合中文用户直觉。
4. 在没有微信小程序主体限制的前提下，尽快获得真实试玩反馈。
5. 初期尽量用 Cloudflare 全栈方案完成上线，减少首版运维面。

### 2.2 V0.1 核心指标

| 指标 | 目标 |
| --- | --- |
| 单局完成率 | >= 40% |
| 平均每局有效猜词次数 | 8-25 次 |
| 连续游玩 | 内测用户平均完成 >= 2 局 |
| 评分合理性 | 人工样本命中预期区间比例 >= 80% |
| 首屏加载 | 移动端 3 秒内可操作 |
| 评分响应 | 90% 猜词请求 <= 2 秒 |
| 单局 AI 成本 | 按真实模型单价和实测 token 计算 |

当前默认评分模型：

1. `deepseek-v4-flash`
2. 默认使用非思考模式
3. 通过 Cloudflare AI Gateway 调用

## 3. V0.1 范围

### 3.1 本期必做

1. 移动端 H5 首页。
2. 匿名会话创建和恢复。
3. 随机开局。
4. 游戏页输入猜词并展示百分比。
5. 猜测历史、当前最高分、有效猜词次数。
6. 猜中结果页。
7. 放弃查看答案。
8. 单局缓存、全局缓存、基础日志、AI 调用观测。
9. 评分反馈入口。

### 3.2 本期不做

1. 微信小程序。
2. 微信登录和网页授权。
3. 每日挑战。
4. 好友挑战。
5. 排行榜。
6. 分享战绩图。
7. 付费、广告、道具。
8. 可视化管理后台。

## 4. 用户与身份

### 4.1 目标用户

| 用户类型 | 需求 |
| --- | --- |
| 轻度小游戏用户 | 快速进入、规则简单、反馈直接 |
| 文字游戏用户 | 能用语义推理推进答案 |
| AI 兴趣用户 | 想体验模型判断语义接近度的效果 |

### 4.2 身份方案

V0.1 使用匿名会话：

1. 首次访问时后端创建 `visitor_id` 和 `session_token`。
2. 前端把 `session_token` 保存到浏览器安全存储中。
3. 后端用 `visitor_id` 关联游戏记录。
4. 清理浏览器数据后，用户视为新访客。

后续版本可以增加微信网页授权或账号体系，但不能影响匿名游玩。

## 5. 页面与流程

### 5.1 总流程

```text
打开 H5
  ↓
创建或恢复匿名会话
  ↓
首页
  ↓
开始一局
  ↓
游戏页反复猜词
  ↓
猜中 / 放弃 / 过期
  ↓
结果页
```

### 5.2 首页

页面目标：让用户立即开始一局。

模块：

| 模块 | 说明 | 优先级 |
| --- | --- | --- |
| 产品标题 | 展示产品名和短说明 | P0 |
| 开始一局 | 创建随机游戏 | P0 |
| 玩法说明 | 输入词，只看接近百分比 | P0 |

### 5.3 游戏页

页面目标：承载核心猜词循环。

模块：

| 模块 | 说明 | 优先级 |
| --- | --- | --- |
| 输入框 | 输入猜词 | P0 |
| 提交按钮 | 发起评分请求 | P0 |

模块：

| 模块 | 说明 | 优先级 |

| 本局状态 | 猜中、放弃、过期 | P0 |

模块：

| 模块 | 说明 | 优先级 |
| --- | --- | --- |

| 单条删除 | 用户主动删除某一局历史 | P1 |
| 全部删除 | 用户主动清空全部历史 | P1 |

## 6. 游戏规则

1. 每局由后端固定一个答案，前端在结束前不能获得答案。
2. 标准答案或显式别名命中时，后端直接返回 100 分并结束游戏。
3. 普通猜词由后端调用 AI 评分。
4. 前端只展示最终分数，不展示 AI 解释。
5. 重复猜词返回已有结果，不增加有效次数。
6. 输入非法、敏感词、限流、AI 超时和系统错误都不增加有效次数。
7. 单局最多 100 次有效猜词；第 100 次仍未猜中时，本次计数后游戏过期。
8. 从创建时间起 24 小时未完成的游戏自动过期。

## 7. 状态机

| 状态 | 含义 |
| --- | --- |
| `playing` | 游戏进行中 |
| `success` | 已猜中 |
| `give_up` | 用户放弃 |
| `expired` | 达到次数上限或超过时间 |

| 当前状态 | 事件 | 下一个状态 | 是否计入有效次数 |
| --- | --- | --- | --- |
| `playing` | 普通有效猜词 | `playing` | 是 |
| `playing` | 标准答案或显式别名 | `success` | 是 |
| `playing` | 重复猜词 | `playing` | 否 |
| `playing` | 非法输入或敏感词 | `playing` | 否 |
| `playing` | AI 超时或系统错误 | `playing` | 否 |
| `playing` | 用户放弃 | `give_up` | 否 |
| `playing` | 第 100 次有效猜词后仍未命中 | `expired` | 是 |
| `playing` | 超过 24 小时 | `expired` | 否 |
| 已结束状态 | 继续猜词 | 原状态 | 否 |

## 8. 埋点

| 事件 | 时机 | 核心字段 |
| --- | --- | --- |
| `session_created` | 首次匿名会话创建 | `visitor_id` |
| `session_restored` | 恢复匿名会话 | `visitor_id` |
| `game_created` | 创建游戏 | `game_id`, `answer_id` |
| `guess_submitted` | 有效猜词 | `game_id`, `guess_id`, `score`, `source` |
| `guess_reused` | 重复猜词 | `game_id`, `normalized_guess` |

| `game_give_up` | 放弃 | `game_id`, `guess_count` |

| `score_feedback_submitted` | 评分反馈 | `game_id`, `guess_id` |

## 9. 验收标准

### 9.1 功能验收

1. 新访客打开页面后能创建匿名会话。
2. 刷新页面后能恢复未结束游戏。
3. 用户能开始随机局并提交猜词。
4. 猜词结果包含 0-100 分数和历史记录。
5. 标准答案或显式别名命中后进入结果页。
6. 放弃后展示答案，并禁止继续猜词。
7. 重复猜词不增加有效次数。

### 9.2 体验验收

1. 移动端首屏 3 秒内可操作。
2. 猜词时有明确加载状态。
3. 网络错误不会清空当前页面状态。
4. 结果页可以直接开始下一局。

### 9.3 评分验收

评分验收以 [05-scoring-spec.md](05-scoring-spec.md) 的人工样本集为准。

### 9.4 数据分析验收

行为分析和评分质量评估以 [08-analytics-and-quality.md](08-analytics-and-quality.md) 为准。V0.1 内测前至少要能产出每日行为漏斗、评分反馈列表、AI 调用成本和答案维度质量排行。
